이번 버전의 기준:
- 카메라가 정상 작동하던 office_face_sampler_final 버전을 기준으로 복구했습니다.
- 카메라 관련 로직은 기존 정상 버전의 방식으로 되돌렸습니다.
- 오디오만 iPhone 대응 방식으로 수정했습니다.

오디오 수정:
- assets/sound1.mp3 ~ sound5.mp3 영문 파일명 사용
- HTML에 audio 태그 미리 배치
- 관상 분석하기 버튼 클릭 즉시 play()
- 3초 뒤 play() 재호출 없음
- 각 mp3 안에 짧은 찰칵음 + 약 3초 대기 + 실제 멘트 포함

Windows Chrome/Edge 카메라 수정 버전:
- 오디오 관련 파일과 로직은 유지했습니다.
- 카메라 요청을 video:true로 단순화했습니다.
- Windows Chrome/Edge에서 오류 원인을 확인할 수 있게 오류명, 감지된 카메라 수, 현재 주소를 표시합니다.

다른 PC에서 카메라가 안 켜질 때 확인:
1. GitHub 저장소 화면이 아니라 GitHub Pages 주소로 직접 접속했는지 확인
   예: https://사용자명.github.io/저장소명/
2. Chrome/Edge 주소창 왼쪽 자물쇠 아이콘 → 사이트 설정 → 카메라 → 허용
3. Windows 설정 → 개인정보 및 보안 → 카메라
   - 카메라 액세스 켬
   - 앱에서 카메라에 액세스하도록 허용 켬
   - 데스크톱 앱에서 카메라에 액세스하도록 허용 켬
4. Zoom, Teams, 카메라 앱 등 카메라를 쓰는 앱 종료
5. 테스트 링크 뒤에 ?v=400 붙이기


NotReadableError 대응 버전:
- 오디오는 변경하지 않았습니다.
- 카메라 선택 박스를 추가했습니다.
- 기본 카메라가 NotReadableError로 실패하면 감지된 다른 카메라를 순서대로 자동 재시도합니다.
- 사용자가 직접 카메라를 선택해서 다시 시도할 수 있습니다.

Windows에서 NotReadableError가 계속 뜨면:
1. Zoom, Teams, 카메라 앱, OBS, 브라우저의 다른 탭을 모두 닫기
2. 작업 관리자에서 Teams/Zoom/Camera 관련 프로세스 종료
3. Windows 설정 → 개인정보 및 보안 → 카메라
   - 카메라 액세스 켬
   - 앱에서 카메라에 액세스하도록 허용 켬
   - 데스크톱 앱에서 카메라에 액세스하도록 허용 켬
4. 장치 관리자 → 카메라 → 사용 안 함/사용 또는 드라이버 업데이트
